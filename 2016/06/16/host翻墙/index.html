翻墙教程(使用hosts文件)

Jun 16, 2016 | 其他 | Hits

概述

本文将介绍如何使用hosts文件翻墙

DNS简介

DNS就是域名系统（domain namesystem).
我们访问互联网上的主机其实是通过ip地址定位的，ip地址是由4个无符号整数组成的，比如192.168.1.1等。由于ip地址不容易记忆，所以诞生了方便人们记忆的域名（网址）。DNS就是把类似与www.google.com这样的域名翻译成ip地址的系统。当我们浏览网页时，浏览器会先访问DNS服务器，根据域名得到ip地址，然后通过ip地址找到相应的网站服务器。

hosts文件

Hosts 文件本来是用来提高解析效率。在进行 DNS 请求以前，系统会先检查自己的 Hosts 文件中是否有这个地址映射关系，如果有则调用这个 IP 地址映射，如果没有再向已知的 DNS 服务器提出域名解析。也就是说 Hosts 的请求级别比 DNS 高。当你的 Hosts 文件里面有对应的 IP 时，它就会直接访问那个 IP，而不用通过 DNS。所以，当我们直接将 Google、Twitter、Facebook 之类的 IP 放入 Hosts 文件后，就可以跳过 DNS 的解析这一步，直接就行 IP 访问，不受 GFW 的 DNS 污染干扰了。
windows的hosts文件在C:\Windows\System32\drivers\etc\目录下
linux的hosts文件在/etc/ 目录下

获取google、facebook等的ip地址

google的ip地址经常变，不过不用担心，我天朝的热心网友会及时收集google、fackbook等互联网厂商的ip地址，并且制作成hosts问件，方便我们的使用。
点击此处下载hosts文件
windows用户下载好hosts文件后，替换C:\Windows\System32\drivers\etc\hosts文件即可
linux用户替换系统的 /etc/hosts文件
linux下最好替换后在最后加一行
127.0.0.1 电脑主机名（不是用户名）不然会影响sudo的速度

替换后可能仍然不能使用google，可能时由于google的香港域名变更的缘故，可以先访问其英文的域名，https://www.google.com/ncr 使用一段时间后会自动定位至香港
