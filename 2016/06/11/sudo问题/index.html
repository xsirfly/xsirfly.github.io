ubuntu16.04 sudo执行命令时command not found解决方法

Jun 11, 2016 | 编程 | Hits

问题描述

前两天安装了ubuntu16.04的系统，今天安装配置java时发现，在/etc/profile下配置好环境变量后，直接使用java命令时可以的，但是使用sudo后却出现了sudo:java command not found问题。

问题原因

在网上找了半天的解决办法后发现原因是：ubuntu 16.04编译sudo包时开启了–with-secure-path选项
–with-secure-path选项的说明：

–with-secure-path[=PATH]
Path used for every command run from sudo(8). If you don’t trust the people running sudo to have a sane PATH environment variable you may want to use this. Another use is if you want to have the “root path” be separate from the “user path.” You will need to customize the path for your site. NOTE: this is not applied to users in the group specified by –with-exemptgroup. If you do not specify a path, “/bin:/usr/ucb:/usr/bin:/usr/sbin:/sbin:/usr/etc:/etc” is used.

因为当在Linux下用sudo执行某一命令时，是在原进程（parent process）的基础上fork出来一个子进程（childprocess），这个子进程是以root权限执行的。然后在子进程中，执行你在sudo后面跟的命令。在子进程中是无法调用涉及到父进程的状态的一些命令的，所以非系统内置命令会被拒绝。这就是为什么会出现command not found的提示。

解决方案

  1. 在/etc/sudoers文件内增加这么一行:Defaults secure_path=”/bin:/usr/bin:/usr/local/bin:…”, 把要用的命令path包括进去。
  2. 用命令的绝对路径。
  3. 使用sudo的env选项,像这样sudo env PATH=$PATH cmd.sh。
  4. 重新编译sudo,不带–with-secure-path选项了.(终极解决办法)。

问题解决

上面的方案中，每次都要修改/etc/sudoers文件貌似太麻烦了，果断不用，而我要是愿意用绝对路径这个方法，肯定就不会折腾了，重新编译sudo好像比较困难，最后我采用了方法3和别名的方法，在/etc/profile 文件的最后加入

source /etc/profile 后就成功了！！！
